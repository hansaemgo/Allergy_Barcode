## Executive Summary

- **핵심 타겟:** 아나필락시스 환아를 둔 **워킹맘 B2C 5,000가구**(SOM-1), 배식 사고 리스크를 안고 있는 **프리미엄 보육기관 B2B 300곳**(SOM-3), 그리고 법적 사각지대에 놓인 **희귀 알러지 환우**(SOM-2, 데이터 인프라 전담).
- **핵심 Job:**
  - **"0.5초 만에 내 아이가 응급실에 갈 간식인지 판별해 주세요"** (김지윤 — B2C, DOS 3.0)
  - **"교사 오배식 한 건으로 폐원되는 재앙을 시스템으로 원천 차단해 주세요"** (박현진 — B2B, DOS 3.6)
  - **"식약처 22종 밖의 희귀 성분을 내가 직접 등재하고 경고할 수 있게 해주세요"** (유나비 — Extreme, DOS 2.8)
- **핵심 Value Proposition:**
  "부모에게는 **0.5초 생사 판별 렌즈**를, 보육 기관장에게는 **배식 사고 ZERO 방어 시스템**을, 희귀 환우에게는 **내가 만드는 생명 위키**를 — 하나의 NLP 코어 엔진 위에서 통합 제공하는 **'생명 방어 플랫폼'**"
- **차별화:** 기존의 '맘카페 수동 질문', '형광펜 수기 식단표', '개인 엑셀 블랙리스트'와 달리, **식약처 100만 건 DB + 크라우드소싱 사설 위키**를 결합한 실시간 NLP 매칭 엔진, **다중 프로필(다자녀) 동시 판별**, **원장-교사-학부모 3중 실시간 통신망**을 통합 제공
- **MVP 포커스:** 1) **0.5초 바코드/OCR 스캔 + 녹색/적색 햅틱 UI**, 2) **B2B 기관 다중 원아 매칭 관리자 Web**, 3) **희귀 성분 크라우드소싱 제보 위키**

## TAM–SAM–SOM 및 Segment 요약

- **정의(요약):**
  - **TAM:** 글로벌 Free-from(대체/안전) 식품 시장 전체 — **약 58조 원 / $43.6B** (CAGR 7.7%)
  - **SAM:** 국내 0~12세 소아 알러지 35만 가구 + 전국 3만 보육기관의 안심 소비/구독 유효 시장 — **약 2,208억 원 / $166M**
  - **SOM (1년 차 초기 거점):** B2C 5,000가구 커머스 수수료 + B2B 300곳 라이선스 — **약 1.98억 원 / $150K**
- **세그먼트 하이라이트:**
  - **Segment A (Q2 — 맹목적 보호망):** 김지윤·최수안 → SOM-1 핵심 B2C 캐시카우 (연 9,000만 원)
  - **Segment C (Q4 — 시스템 방어자):** 박현진·송미정 → SOM-3 B2B 안정적 구독 (연 1.08억 원)
  - **Segment B (Q1 — 능동적 생존자):** 이도윤·유나비 → SOM-2 전문성 검증 + DB 해자
  - **Segment D (Q3 — 가치 소비군):** 강태민·정하늘·이건우 → **타겟 후순위 보류** (DOS ≈ 0)
- **SOM 시뮬레이션 산출 근거:**
  1. B2C: 5,000가구 × 월 3만 원 추천 결제 × 5% 수수료 × 12개월 = **연 9,000만 원**
  2. B2B: 300개 원 × 월 3만 원 라이선스 × 12개월 = **연 1.08억 원**
  3. **총 SOM: 연 1.98억 원 ($150K)**

## AOS-DOS 전략 매트릭스 핵심 요약

AOS(순수 고통 크기)와 DOS(시장성 조정 후 수익 기회)의 교차 분석에서 도출된 4개 구역별 전략입니다.

| 구역 | DOS 범위 | 대상 페르소나 | 전략 |
| :--- | :--- | :--- | :--- |
| 🚨 **1차 PMF 폭발 (Cash-Cow)** | DOS ≥ 3.0 | **박현진**(3.6), **김지윤**(3.0) | 마케팅·영업 예산 100% 최우선 집중 투입 |
| 🛠️ **2차 신뢰 해자 (Moat)** | DOS 2.0~2.9 | **유나비**(2.8), **송미정**(2.4) | 수익보다 크라우드소싱 DB·레퍼런스 확보에 투자 |
| 📉 **3차 보조 확장** | DOS 1.0~1.9 | **신영숙**(1.8), **최수안**(1.6), **이도윤**(1.4) | 보호자 대리 연동, 가족 종속 등 우회 채널로 서비스 |
| 🛑 **4차 타겟 폐기** | DOS < 1.0 | **강태민**, **정하늘**, **이건우**, **윤자영** | 기존 대안에 만족 → 초기 마케팅 예산 절대 투입 금지 |

> **핵심 인사이트:** AOS에서 공동 1위였던 **유나비(4.0)**가 DOS에서 3위(2.8)로 하락하고, AOS 3위였던 **김지윤(3.0)**이 DOS에서 2위(3.0)로 상승합니다. 이는 '고통의 크기'와 '돈이 되는 시장'이 반드시 일치하지 않음을 보여주며, **MR(시장 배수) 보정이 없었다면 MVP 자원 배분이 잘못되었을 것**임을 의미합니다.

  - **정보 해독 장벽:** 깨알 같은 원재료·교차오염 텍스트를 육안으로 읽어야 하는 물리적 고통

  - **시간 전쟁:** 퇴근 후 15분, 점심 배식 전 30분 등 극도로 압축된 의사결정 시간
  - **다중 필터 인지 과부하:** 여러 자녀/원아의 서로 다른 항원을 동시 연산하는 뇌의 한계
  - **디지털 소외:** 노안, 폰 조작 공포, 복잡한 뎁스가 있는 UI에 대한 원초적 거부
- **CJM 단계별 대표 Pain:**
  1. **인식·탐색:** 맘카페 답변 30분 지연, 어느 정보가 최신/공식인지 판별 불가
  2. **의사결정:** "이 앱이 내 아이 목숨을 걸만큼 정확한가?" — 데이터 신뢰율 99.9% 어필 필요
  3. **사용:** 0.5초 내 결과 미제공 시 즉각 이탈, 텍스트 위주 UI 시 노인/10대 이탈
  4. **유지:** "월 4,900원이 대수인가" vs "돈 낭비 아닐까" — 방어 기회비용 리포트로 정당화
  5. **Lock-in:** B2B 해지 시 학부모 이탈 폭발, B2C는 '이 앱 없이는 장보기 불가' 의존 형성

| **R8** | **식약처 DB의 구조적 한계 (데이터 정합성)** | 식약처 의무표기 22종 이외의 교차오염 원료는 **표시 의무가 없어 DB에 아예 존재하지 않음** → 앱이 '안전'이라 판단했지만 실제로는 미표기 교차오염이 존재하는 '사각지대' 품목 다수 | **확실** | **높음** | ①'교차오염 가능성' 표기가 있는 제품은 **무조건 노란불(주의)** 처리 → 보수적 판별 원칙 ②크라우드소싱(F4)을 통해 사용자 제보 데이터로 사각지대를 점진적으로 보완 ③중기적으로 제조사 직통 API 제휴 추진(제조사가 자발적으로 교차오염 라인 정보 공개) | ①사각지대 품목 비율이 전체의 20% 이상이면 결과 화면에 **"이 제품은 교차오염 정보가 불완전합니다. 제조사 확인을 권장합니다."** 경고 문구 상시 노출 ②장기적으로 **식약처에 '교차오염 전수 표기 의무화' 정책 제안서** 제출 (업계 리더십 확보) |

## (추가) 비활성 사용자(Non-User)의 생태계 기여 구조

기존 VPS 프레임워크에서 간과하기 쉬운 **"앱을 거부하지만 앱 생태계에 편입되는"** 비활성 사용자(김성철 삼촌, 윤자영 안아키맘)의 가치를 별도로 분석합니다.

| 비활성 사용자 | 앱 설치 | 매출 기여 | 실제 생태계 기여 |
| :--- | :--- | :--- | :--- |
| **김성철** (아날로그 친척) | ✖ (카톡 링크만 소비) | 0원 | 부모(김지윤)가 보낸 **카톡 Web AR 스캐너 링크**를 통해 '과자가 위험하다'는 사실을 체감 → 이후 앱 인증 완료된 안전 간식 리스트에서만 주문 → **부모의 갈등 비용 0으로 만드는 호신용 도구**로서 앱의 해지 불가 가치를 극대화 |
| **윤자영** (자연주의 맘) | ✖ (웹 익명 열람만) | 0원 | 유기농 공장 교차오염 결함을 서면으로 받아내어 웹 위키에 익명 제보 → **가장 까다로운 비가공식품 '현장 데이터 인부(Data Worker)'로 자발적 종속** → 앱 DB 품질 폭발적 상승 |

> **시사점:** LTV(매출)가 완전히 0원인 사용자조차 **데이터 품질 기여·갈등 해소·생태계 파이 확장**이라는 비재무적 가치를 통해 플랫폼의 핵심 인프라로 기능합니다. 예비창업자는 "돈 안 되는 유저 = 의미 없는 유저"라는 단순한 등식에서 벗어나, **플랫폼 전체의 네트워크 효과와 데이터 해자에 기여하는 모든 행위자**를 생태계 설계에 포함시켜야 합니다.
