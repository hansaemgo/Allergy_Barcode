## 1. JTBD 요약 카드 (Persona-별 인터뷰 검증 결과)

### 💡 타겟 1: 김지윤 (30대 고불안 워킹맘)
| 구분 | 내용 |
| --- | --- |
| **Persona / Segment** | **코어 대상 (B2C)** – 쫓기는 시간과 아이 생명 불안에 시달리는 30대 워킹맘 |
| **Situation** | **퇴근 직후 피로에 절어 대형마트에서 내일 어린이집에 보낼 간식을 골라야 하는 15분** |
| **Job Statement** | **아이가 먹고 응급실에 가지 않을 가장 안전한 간식을 스트레스 없이 0.5초 만에 고르고 싶다.** |
| **Desired Outcome** | **성분 확인 시간 1분 → 0.5초 단축, 교차오염 판별 오류 0건, 선택 가능한 안심 간식 수 3배(3개→10개) 증가** |
| **4 Forces** | **Push:** 깨알 같은 텍스트 판독 피로, 맘카페 답변 지연의 짜증<br>**Pull:** 기계적이고 극단적인 O/X 햅틱 스캐너의 확실성<br>**Habit:** 결국 귀찮음에 이기지 못하고 늘 먹던 안전한 3개 과자만 반복 구매함<br>**Anxiety:** 식약처 DB 업데이트 지연으로 앱이 안전하다고 오작동할까 봐 두려움 |
| **Current Solutions** | **맘카페 질문 글 업로드 후 답변 대기 + 성분 라벨 사진 줌인 판독** |
| **Switch Triggers / Barriers** | **Trigger:** 아이가 매일 똑같은 간식에 질려 떼를 쓰며 엄마의 죄책감을 극대화했을 때<br>**Barrier:** 앱 실행 후 로그인/배너 광고 등 스캔에 도달하기까지의 조작 뎁스(Depth) |
| **Evidence (근거)** | "바빠 죽겠는데 알쏭달쏭한 성분명 찾아보고 있을 시간이 없어서 결국 먹던 것만 사요."<br> "앱이 괜찮다고 했는데 애가 아프면 누구 핑계를 대야 하나요?" |
| **Priority (AOS / DOS)** | **AOS = 3.0**, **DOS = 3.0** (Importance=5, Satisfaction=2, MR=1.0) |
| **Notes (추모 메모)** | **MVP 필수 요건:** 앱 진입 시 초기화면 렌더링 소거, 0.5초 내 '녹색/적색' 햅틱 피드백.<br>**위험 헷징:** 오탐지 시 진료비 보상 등 법적 책임 배상 보험 제휴 마케팅 필요. |

### 💡 타겟 2: 박현진 (50대 프리미엄 어린이집 원장)
| 구분 | 내용 |
| --- | --- |
| **Persona / Segment** | **코어 대상 (B2B)** – 월 1,000만 원 이상의 운영 예산을 만지는 100명 규모 보육 원장 |
| **Situation** | **매일 낮 12시 30분, 온 교사진이 지친 상태로 전 원아에게 식판을 배분해야 하는 급발진 상황** |
| **Job Statement** | **오배식으로 인한 학부모 소송과 폐원을 원천 차단하여 내 사업장을 안전하게 보호하고 싶다.** |
| **Desired Outcome** | **배식 휴먼 에러 0건 유지, 학부모 알러지 컴플레인 0건 달성, 스태프 인지 노동 파괴** |
| **4 Forces** | **Push:** 보육교사의 그날그날 컨디션과 육체 피로에 기댄 아슬아슬한 수동 대조 시스템<br>**Pull:** 바코드 스캔 한방에 원장/학부모/교사 폰으로 통신되는 3중 방어막의 공신력<br>**Habit:** 익숙한 종이 식단표 벽 부착 및 구두(말로) 크로스체킹 관행<br>**Anxiety:** 원 단위의 SaaS 월 구독료(10만원 이상) 및 보육교사들의 새로운 디지털 툴 학습 강력 거부감 |
| **Current Solutions** | **식단표 현관 부착 + 영양사의 형광펜수기 체크 + 담임교사 구두 전달** |
| **Switch Triggers / Barriers** | **Trigger:** 옆 지역 어린이집에서 땅콩 오배식 사고로 맘카페에 고발당해 폐쇄 위기라는 소문을 들었을 때<br>**Barrier:** 교사용/조리사용 앱 단말기 추가 보급 비용 및 UX 장벽 |
| **Evidence (근거)** | "사람이 하는 거라 매일 시한폭탄 안고 점심시간을 넘겨요."<br> "우리 원의 '경쟁 무기(세일즈 포인트)'가 될 수 있다면 구독료 10만 원은 저렴하죠." |
| **Priority (AOS / DOS)** | **AOS = 4.0**, **DOS = 3.6** (Importance=5, Satisfaction=1, MR=0.9) |
| **Notes (추모 메모)** | **MVP 필수 요건:** 교사용 모바일 UX는 카카오톡 수준으로 쉬어야 함. 학부모 대상 'SafeBite 도입 안심보육기관' 현판/인증마크 실물 리워드 지원 필수. |

| **Situation** | **국가 DB 22종에 없는 식품 제조사의 '숨은 동물성/비건/희귀 2차 오염 원료'를 본인이 직접 캐내었을 때** |

| **Desired Outcome** | **내가 팩트체크한 내용이 1시간 내 정식 모수에 등재, 커뮤니티 조회수/도움 추천 100건 피드백** |

| **Current Solutions** | **고객센터 녹취 후 개인 구글 스프레드시트에 기입하여 톡방 공유** |

## 2. 인터뷰 결과 → Outcome 목록 표 (우선순위 최종 결론)

| :--- | :---: | :---: | :---: | :---: | :---: | :--- |

>💡 **전략적 시사점:**
>가상 인터뷰 결과를 분석해보면 `AOS/DOS 분석`에서 수치상으로 보았던 가치들이 "시간 단축", "학부모 안심 마케팅", "영웅주의 피드백"이라는 구체적인 기능적 산출물(Outcome)로 확인되었습니다. 특히 지불 능력이 가장 높은 프리미엄 어린이집 원장 타겟의 니즈는 단순한 '스캐너 기능'이 아니라 원의 **마케팅 무기 및 법적 방어막(보험)** 역할을 해줄 시스템에 있음을 알 수 있습니다. 이는 MVP 기획의 최우선 순위로 고려되어야 합니다.
